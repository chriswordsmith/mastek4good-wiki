# Problem 6

System Design Ideas & Instructions

* Unified system — multi-modal data input; moment data summarisation via LLM; agent uses summary to update and suggest/revise plan; memory + DB; tracks real-time info (vitals, interventions, timings, photos); informs support planning; accessible
* Ensure new system is artificially intelligent to: access/process all relevant data (e.g. patient record & pathway); understand and confirm staff conversation/input by natural language
* Intelligent inference from natural conversations (voice recording, transcripts, etc.)
* Codewords — could reduce repeated exposure to traumatic details


Questions — Existing System & Data

* What is actually difficult with the existing system?
* What data sets need to be collected?
* How much time does it take to get data from the field to final state in the system? How long should it take?
* What happens to historic support plans?
* Access to previous records — BUT need to know?


Questions — User Experience & Interaction

* What does good look like?
* How do we best avoid repeat questions, i.e. avoid re-living traumatic situations?
* What is a natural way to record conversations? How do we keep the system out of the conversation?
* Will video recording help in capturing the questions?
* What tools/pages are workers familiar with — could we copy components from these to create a page/view?


Questions — Integration & Risk

* Can we interface with other service providers? What does this look like?
* What risks are we talking about?
